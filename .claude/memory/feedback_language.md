---
name: Language policy for project artifacts
description: All project files, git commits, code comments, and repo content must be in English
type: feedback
---

Everything inside the git repository must be in English: commit messages, code comments, Doxygen docs, README, issue/PR text, and any other project file.

**Why:** The project is an English-language open-source repo. Mixing languages in commits and source makes history and collaboration harder. Hungarian is fine for conversation with the user, but never in the repo itself.

**How to apply:** Always write commit messages, code comments, and documentation in English. The only exception is user-facing program output text, and only if the program explicitly supports multiple languages or a specific Hungarian locale — which it currently does not.
